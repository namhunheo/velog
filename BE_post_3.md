### 1. 프로그래밍 종류
> 
**1.1. 절차지향 프로그래밍**
: 각 명령에는 순서가 있고, 반드시 한 방향으로 수행되며 동작
: 대표적으로 c언어가 있으며, 실행 속도가 빠르지만, 유지보수와 디버깅이 어려움

![](https://velog.velcdn.com/images/huni6711/post/e2ff9ef0-0835-4f2e-a89f-57230e7b40e2/image.png)^절차지향 vs 객체지향

>**1.2. 객체지향 프로그래밍**
: 어떤 사물의 도면에 해당하는 'class'를 정의하고 이 클래스를 실체화한 'object'를 만들어 객체 간의 상호작용을 통해 프로그램을 만듦
: 주요 특징 - 추상화, 캡슐화, 상속
>>- **추상화**
: 공통 속성과 기능을 추출해 이름을 붙이는 과정으로, 클래스를 정의하는 것
: ex) '새' 클래스 : 속성 - 날개, 부리 / 기능 - 날기, 먹기 / 객체 - 비둘기, 참새 등
>>
- **캡슐화**
: 속성과 그 속성을 조작하는 기능(함수)를 하나의 캡슐로 묶는 것
: 외부로 노출하고 싶은 속성이나 기능을 제어하기 위함.
- **상속**
: 기존 상위 클래스의 속성과 기능을 가져다 하위 클래스에서 재사용하는 것
: ex) 새가 날기, 먹기만 하다가 뛰는 새가 발견된다면 '뛰는 새'로 클래스 정의하게 되는데, 그럼 '새'라는 속성과 기능을 가져오는 상속을 함.

>**1.3. 함수형 프로그래밍**
: 특징 - 불변성, 선언형

> - 추가 정보
	1. **자바스크립트(JS)**
    	: 웹 페이지의 정적인 HTML 콘텐츠를 동적으로 변경하거나 이벤트와 같은 상호작용을 구현하는 인터프리터* 언어
    : 데이터 명시가 필요없는 동적 타입 언어
    : HTML, CSS를 동적으로 다룰 수 있고, DOM API를 기반으로 웹 브라우저 내의 유저와의 상호작용 처리 가능
    2. **타입스크립트(TS)**
    : JS의 단점을 보완하기 위한 컴파일* 언어
    : JS의 모든 기능을 유지하면서 정적 타입과 관련된 문법을 추가
    3. **자바(JAVA)**
    	1) 자바 가상 머신(JVM)을 통해 실행
        2) 숫자(int, float 등)와 논리 값(true, false)을 제외한 모든 데이터를 객체로 구성
        3) 모든 메모리에의 접근을 자바 시스템이 관리하고 제한
        4) 멀티스레드* 쉽게 구현 가능

### 3. 백엔드 프레임워크
> 3.1 라이브러리란?
: 개발하는데 필요한 기능을 미리 구현해놓은 코드의 모음, 재사용하기 위해!
: 개발자가 코드의 흐름을 직접 제어 -> 필요에 따라 라이브러리 호출해서 사용
> 3.2. 프레임워크란?
: 복잡한 문제를 해결하거나 서술하는데 필요에 따라 가져다가 사용되는 기본 구조로, 일정한 형태와 기능을 제공하는 틀.
: 라이브러리와는 다르게, 주어진 코드의 속에서 프로그래밍해야함.


|      프레임워크      |    주요 언어   | 핵심 특징                              | 장점                     | 단점                      |
| :-------------: | :--------: | :--------------------------------- | :--------------------- | :---------------------- |
|  **Express.js** | JavaScript | 경량 웹 서버 / 미들웨어 중심 구조               | 단순하고 빠름, 자유도 높음        | 구조화 부족, 대형 프로젝트엔 관리 어려움 |
|    **NestJS**   | TypeScript | Angular 철학 기반의 **모듈형 아키텍처**, DI 지원 | 유지보수 쉬움, 대규모 서비스 적합    | 러닝 커브 존재                |
|    **Spring**   |    Java    | 엔터프라이즈급 **IoC, AOP** 기반 프레임워크      | 확장성 우수, 안정성 높음         | 설정 복잡, 초기 구성 어려움        |
| **Spring Boot** |    Java    | Spring 자동화 / 내장 Tomcat으로 즉시 실행     | 설정 단순, 빠른 개발 속도        | 세밀한 제어 어려움              |
|    **Django**   |   Python   | **MTV 패턴** 기반 풀스택 프레임워크            | 관리자 페이지 자동 생성, 빠른 개발   | 유연성 낮음, 무겁다             |
|    **Flask**    |   Python   | 경량 마이크로 프레임워크 / 최소 구조              | 단순하고 커스터마이징 용이         | 기능 직접 추가 필요, 확장성 한계     |
|   **FastAPI**   |   Python   | **비동기 지원 + 타입 힌트 기반** 현대적 프레임워크    | 속도 빠름, 문서 자동화(Swagger) | 러닝 커브 약간 있음             |



> - 용어 설명
1. **인터프리터** 언어
: 소스 코드를 한 줄씩 읽고 즉시 기계어로 변환하여 실행, 모든 코드가 실행될 때까지 코드를 읽고 실행하는 작업을 반복
2. **컴파일** 언어
: 전체 소스 코드를 읽고 분석한 후, 한번에 기계어로 변환(컴파일)하여 그 결과물인 실행파일을 저장해 실행.
: 만약, 소스 코드를 수정하면 전체 소스 코드를 다시 컴파일한 후 실행
3. 멀티스레드
: 하나의 프로세스 안에서 여러 작업(스레드)을 동시에 수행하는 기술
→ 즉, CPU 자원을 병렬로 활용해 성능을 높이고 응답 속도를 개선
