### 1. 클라우드 컴퓨팅

>
- 언제 어디서든 인터넷을 통해 접속할 수 있는 컴퓨팅 환경으로, 
**<클라우드>**라고도 함.
- 데이터, 소프트웨어, 컴퓨팅 자원 등을 인터넷을 통해 제공받아 사용하는 형태인  클라우드가 도입되기 전에는 개인이나 회사가 자체적으로 서버를 구축하고 운영하는 온프레미스(ON-premise) 시스템을 사용했으나, 이젠 클라우드가 보급되며 자체 서버를 보유하지 않고 자원을 대여해서 사용!

> **1.1 온프레미스(ON-Premise)**
: 사용자가 직접 컴퓨팅 자원을 보유하고 관리하는 방식
: 자체적으로 서버, 스토리지 등을 구입, 설치 후, 유지보수까지ㅜㅠ
- 특징
	1. 사용자가 보안과 데이터 소유권을 직접 통제
     	2. 외부 네트워크 연결없이 내부 네트워크에서 서버 접근 가능
    	3. 초기 투자 비용과 유지보수 비용 큼, 자원의 확장성, 유연성 제한
        
![](https://velog.velcdn.com/images/huni6711/post/85794235-268b-4cff-aec7-b440c2097706/image.png)



>**1.2 클라우드 시스템**
: IT자원을 클라우드 서비스 제공 업체에 의뢰해 관리하는 방식
- 특징
	1. 사용한 자원에 대해서만 비용 지불하므로 초기 비용 적음
    	2. 복잡한 인프라 구축과 유지보수 부담 줄어, 개발자는 서비스에만 집중
        3. 확장성과 유연성이 뛰어나 서비스 규모에 따라 자원을 신속하게 확장하거나 축소 가능
        4. 인터넷에 연결된 어느 장치에서나 접속 가능, 업체가 제공하는 다양한 기능과 서비스도 이용 가능
        
---

>
**1.3 클라우드 서비스 종류**
크게 2가지 기준으로 구분
1. 서비스 제공방식(공용 클라우드, 사설 클라우드, 하이브리드 클라우드)
2.  서비스 제공범위(IaaS, PaaS, SaaS)

> - 서비스 제공방식에 따른 클라우드 서비스
**1. 공용 클라우드 **
    : 다수의 유저가 공유하는 클라우드 환경, 클라우드 서비스 제공 업체가 소유하고 관리
    : EX) 아마존의 AWS, 마이크로소프트의 Azure
**2. 사설 클라우드**
: 단일 조직이나 기업이 사용하는 클라우드 환경, 자체 소유 및 관리
**3. 하이브리드 클라우드**
: 공용 + 사설 클라우드, 자체 운영하면서 필요에 따라 공용 클라우드 자원 사용

![](https://velog.velcdn.com/images/huni6711/post/d3cdead3-726c-40db-9233-d3822ee48516/image.png)

>
- 서비스 제공범위에 따른 서비스
**	1. IaaS(Infra as a Service)**
    : 클라우드 서비스 제공 업체가 가상 인프라 제공하는 모델로,
    플랫폼과 소프트웨어를 지원하고 실행하기 위한 기반 제공
    : ex) AWS, 애저  등
**   2. PaaS(Platform as a Service)**
   	: 웹 애플리케이션 구축 및 실행할 수 있는 인프라와 플랫폼 제공,
    개발자는 코드에만 집중하고, 서버 구성 및 관리 등 신경 X
    : ex) 구글 앱 엔진, 애저 앱 서비스
**    3. SaaS (Software as a Service)**
	: 클라우드 서비스 제공 업체가 아예 웹 애플리케이션을 제공하는 모델
    : 웹앱을 클라우드에서 바로 실행 가능하여 유저는 웹 브라우저에서 작업 수행
    : ex) 구글 워크스페이스, 마이크로소프트365
    
![](https://velog.velcdn.com/images/huni6711/post/01f4a3f9-310e-45c2-88c4-3dabe220ed04/image.png)

### 2. AWS
>
- AWS : 아마존 공용 클라우드 서비스
- 대표적인 서비스로는 EC2, S3, RDS, IAM.

>**2.1 EC2 (Elastic Compute Cloud) : 가상서버 서비스**
: 가상의 서버 컴퓨터를 프로비저닝하고 실행 가능케 하는 것
```
프로비저닝 - 필요 시, 바로 사용 가능하도록 인프라 구성 및 관리 작업을 자동화하는 것
```
>: EC3에서 유저는 쉽게 컴퓨팅 자원을 빌리고, 사용한 만큼만 비용 지불.
: EC2 **인스턴스** - EC2로 만든 서버 객체
> 
- 서버의 용도 종류
1. 웹 서버 - Client로부터 HTTP요청받아 정적인 웹 페이지를 제공하는 서버 / ex) HTML, CSS, JS, 이미지 등 정적인 파일 제공
2. API 서버 - 클라이언트로부터 API요청 받아 응답하는 서버 /
ex) JSON, XML 형식 데이터
3. 프록시 서버 - 클라이언트와 서버 간 중계 역할 수행
4. FTP 서버 - 파일 업로드 및 다운로드 서비스 제공

>
**2.2 S3 (Simple Storage Service) : 스토리지 서비스**
: 파일 저장하는데 사용, 클라우드 상 큰 창고!
: **S3버킷** - S3에는 실제 파일이 보관되는 여러 바구니가 존재
- **특징**
	1. 빈 상태에서 시작
  2 버킷에 보관하는 파일의 개수, 크기, 유형의 제한 없음.
  3 필요한 유저에게만 접근 권한 부여
  4 파일의 안정성 보장하는 다양한 보안 및 암호화 기술 제공
- **실제 S3 활용**
	**1. 웹사이트 호스팅** : HTML, CSS, JS 등 S3버킷에 저장되면 웹 브라우저에서 해당 콘텐츠 사용 가능
    	**2. 미디어 콘텐츠 호스팅** : 웹 애플리케이션의 각종 사진, 음악 등미디어 콘텐츠를 저장 및 관리
       ** 3. 데이터 백업 및 복원** : 데이터 백업 및 복원 솔루션 제공
       **4. 데이터 저장 및 분석** : 머신러닝, 데이터 분석 등 데이터 수집하고 저장
  
![](https://velog.velcdn.com/images/huni6711/post/fce583b1-56a9-40ad-acdb-bf634ffb37c2/image.png)

---
>
**2.3 RDS(Relational Database Service) : 데이터베이스 서비스**
: 관계형 데이터베이스를 구축하고 관리 가능토록 도와주는 클라우드 서비스
: 이미 준비된 DB관리 시스템을 조립해 사용 가능
- **RDS 활용**
**	1. DB생성**
    : 다양한 RDB(MySQL, PostgreSQL 등) 엔진 지원
** 2 데이터 백업 및 복원**
 : 자동 백업 기능을 제공해 일관된 시점의 스냅샷 생성 후, 보관
** 3 DB 모니터링**
 : DB 인스턴스의 성능, 용량, 연결 등 모니터링 기능(CPU, 메모리 등)
 **4 지원 자동 조정**
 : DB 인스턴스의 크기와 용량을 필요에 따라 자동으로 조정 가능
 
---
>
**2.4 IAM(Identity Access Management) : 보안 및 접근 권한 제어**
: AWS 자원에 접근하고 사용 가능한 유저의 권한을 제어
- **IAM 역할**
	**1. 사용자 권한 관리**
    	**2 사용자 역할 관리**
        : AWS자원에 접근하는 서비스 or 특정 유저 계정에 역할 부여(=IAM롤)
        **3 사용자 그룹관리
        4 자원 접근 정책 관리
        5 암호 정책
        6 MFA 관리**
        : 추가 인증 수단(MFA) 지원 -> 보안 강화
